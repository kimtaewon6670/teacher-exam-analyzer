# 4.5 UI 흐름

전체 UI는 MainWindow의 사이드바를 통해 화면을 전환한다.

## 주요 메뉴 흐름

대시보드 → 학생 관리 → 문제은행 관리 → 시험지 생성 → 결과 입력 → 결과 분석

## 기본 사용 흐름

1. 학생 관리에서 학생 정보를 등록한다.
2. 문제은행 관리에서 문제를 등록한다.
3. 시험지 생성에서 문제를 자동 추출하거나 직접 선택한다.
4. 생성된 시험지를 저장하고 PDF로 출력한다.
5. 결과 입력에서 학생 답안을 입력하거나 CSV 파일을 불러온다.
6. 답안을 저장하고 자동 채점을 실행한다.
7. 결과 분석에서 시험 결과를 확인한다.
8. 대시보드에서 전체 요약 결과와 학생별 결과를 확인한다.

## 화면별 역할

### 대시보드

앱 실행 시 주요 시험 결과 요약을 보여주는 화면이다.

표시 내용:

- 전체 시험 결과 요약
- 평균 점수
- 정답률
- 학생별 결과
- 그래프 기반 시각화

### 학생 관리

학생 정보를 등록하고 관리하는 화면이다.

주요 흐름:

1. 학생 이름, 학번, 반 정보를 입력한다.
2. 등록 버튼을 클릭한다.
3. 학생 목록에 등록된 학생이 표시된다.
4. 검색 또는 반 필터를 통해 학생을 조회한다.
5. 필요 시 학생 정보를 수정하거나 비활성화한다.

### 문제은행 관리

단답형 문제를 등록하고 관리하는 화면이다.

주요 흐름:

1. 문제 내용, 유형, 세부 분류, 난이도, 기준 정답, 허용 답안을 입력한다.
2. 문제를 등록한다.
3. 문제 목록에서 등록된 문제를 확인한다.
4. 검색과 필터를 통해 문제를 조회한다.
5. 필요 시 문제를 수정하거나 비활성화한다.

### 시험지 생성

문제은행의 문제를 기반으로 시험지를 생성하는 화면이다.

주요 흐름:

1. 시험명, 연도, 학기, 시험 구분, 대상 반을 입력한다.
2. 문제 추출 조건을 설정한다.
3. 자동 추출 또는 직접 선택으로 문제를 구성한다.
4. 선택된 문제 목록을 확인한다.
5. 시험지를 저장한다.
6. 필요 시 PDF로 출력한다.

### 결과 입력

시험 후 학생 답안을 입력하는 화면이다.

주요 흐름:

1. 시험, 반, 학생을 선택한다.
2. 학생별 답안을 입력한다.
3. CSV 파일을 불러와 답안을 입력할 수도 있다.
4. 입력값을 검증한다.
5. 답안을 저장하고 자동 채점을 실행한다.
6. 학생별/문항별 결과를 확인한다.

### 결과 분석

저장된 결과를 바탕으로 성취도를 분석하는 화면이다.

주요 흐름:

1. 시험과 반을 선택한다.
2. 분석 조회 버튼을 클릭한다.
3. 반 평균 점수, 전체 정답률, 전체 오답률을 확인한다.
4. 문항별 정답률과 오답률을 확인한다.
5. 유형별, 세부 분류별, 난이도별 취약 영역을 확인한다.
