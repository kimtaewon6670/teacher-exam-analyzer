# 7.3 진행 결과

> 작성 기준: `feature/integration` 브랜치의 최종 통합 코드 구조와 기능 흐름

| 영역 | 진행 상태 | 비고 |
| --- | --- | --- |
| 프로젝트 구조 | 구현됨 | MVC 폴더 구조와 `main.py` 실행 진입점 구성 |
| 학생 관리 | 구현됨 | 학생 등록, 조회, 수정, 비활성화 Repository/Controller/View 구성 |
| 문제은행 관리 | 구현됨 | 문제 등록, 수정, 활성/비활성, 검색/필터 흐름 구성 |
| 시험지 생성 | 구현됨 | 자동 추출, 직접 선택, 미리보기, 저장, 선택 초기화 흐름 구성 |
| 시험지 삭제 | 구현됨 | 저장 시험지 삭제 요청을 exam_id 기준으로 처리 |
| PDF 출력 | 구현됨 | `pdf_export_service.py`와 reportlab 기반 PDF 출력 구성 |
| 결과 입력 | 구현됨 | 직접 입력, 학생 변경 시 답안 로드, CSV 업로드/미리보기 구성 |
| CSV 처리 | 구현됨 | 학번 매칭, 문항 번호 헤더 처리, UTF-8 BOM/CP949/EUC-KR 파싱 구성 |
| 자동 채점 | 구현됨 | 기준 정답/허용 답안 비교, exam_id + student_id 기준 교체 저장 구성 |
| 결과 보기 | 구현됨 | 학생별 요약과 문항별 상세 결과 반환 구조 구성 |
| 대시보드 | 구현됨 | 실제 DB 기반 요약, 차트 데이터, 학생별 결과, 전체 보기, CSV 내보내기, 페이지 이동 구성 |
| 분석 화면 | 구현됨 | 정답률/오답률, 취약 유형, 전체 정답 예외 처리 구성 |
| 문서 | 진행 중 | 통합 브랜치 기준 요구사항, 테스트 계획, 구현 일정 문서 정리 중 |
