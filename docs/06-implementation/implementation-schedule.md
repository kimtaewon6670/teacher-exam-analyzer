# 7. 구현 일정

## 7.1 개발 마일스톤

| 단계 | 기간 | 목표 | 산출물 |
| --- | --- | --- | --- |
| M1. 프로젝트 기반 구축 | 1주차 초반 | 폴더 구조, SQLite DB, 기본 모델/Repository 구성 | 프로젝트 구조, DB 초기화 코드, 기본 모델 |
| M2. 학생/문제은행 관리 | 1주차 | 학생 CRUD, 문제 CRUD, 검색/필터 구현 | 학생 관리 화면 연동, 문제은행 관리 기능 |
| M3. 시험지 생성 | 2주차 초반 | 조건 기반 문제 추출, 직접 선택, 미리보기, 저장 구현 | 시험지 생성 Controller/Service, 저장된 시험지 목록 |
| M4. PDF 출력 | 2주차 | 시험지 PDF 출력 및 답안 미노출 처리 | PDF 출력 기능 |
| M5. 결과 입력/CSV | 2주차 후반 | 직접 입력, CSV 업로드, 입력값 검증, 학생별 답안 저장 | 결과 입력 기능, CSV 파싱 |
| M6. 자동 채점/결과 조회 | 3주차 초반 | 답안 정규화, 기준/허용 답안 비교, 결과 저장/조회 | 채점 결과, 결과 보기 |
| M7. 결과 분석/대시보드 | 3주차 | 통계 계산, 차트 데이터, 학생별 결과 표 구현 | 분석 화면, 대시보드 데이터 |
| M8. 테스트 및 문서 정리 | 3주차 후반 | 오류 수정, 테스트 케이스 검증, 문서 완성 | 요구사항 문서, 테스트 결과, 최종 제출본 |

## 7.2 역할 분담

| 담당 | 역할 | 주요 작업 |
| --- | --- | --- |
| UI 담당 | PySide6 화면 구현 | 사이드바, 대시보드, 학생 관리, 문제은행, 시험지 생성, 결과 입력, 결과 분석 화면 구현 |
| Core 담당 | DB 및 핵심 로직 구현 | SQLite DB, Repository, Controller 연결, 시험지 생성, 채점, 분석, CSV/PDF 로직 구현 |
| Docs 담당 | 문서 작성 및 검증 | 요구사항, ERD, 유스케이스, 시퀀스, 테스트 케이스, README 정리 |
