# 5.2 기능별 테스트 케이스

| TC ID | 테스트 항목 | 사전 조건 | 절차 | 기대 결과 |
| --- | --- | --- | --- | --- |
| TC-01 | 학생 등록 | 앱 실행 및 DB 초기화 | 학생 이름, 학번, 반 입력 후 등록 | 학생 목록에 신규 학생 표시 |
| TC-02 | 학생 수정 | 등록된 학생 존재 | 학생 선택 후 이름/학번/반 수정 | 수정된 정보가 목록과 DB에 반영 |
| TC-03 | 학생 비활성화 | 시험 기록이 있는 학생 존재 | 학생 비활성화 실행 | 학생은 비활성화되며 기존 결과는 보존 |
| TC-04 | 문제 등록 | 문제은행 화면 진입 | 문제 내용, 유형, 난이도, 정답 입력 후 저장 | 문제 목록에 신규 문제 표시 |
| TC-05 | 문제 검색/필터 | 여러 유형의 문제 존재 | 유형/난이도/태그 조건으로 검색 | 조건에 맞는 문제만 표시 |
| TC-06 | 시험지 자동 추출 | 문제은행에 충분한 문제 존재 | 유형별 문항 수와 난이도 조건 입력 후 자동 추출 | 조건에 맞는 문제 수만큼 추출 |
| TC-07 | 문제 부족 처리 | 조건에 맞는 문제가 부족 | 자동 추출 실행 | 부족 안내 메시지 또는 가능한 문제만 표시 |
| TC-08 | 직접 선택+자동 추출 저장 | 자동 추출 문제와 직접 선택 문제 존재 | 시험지 저장 실행 | 두 그룹의 문제가 모두 시험지에 저장 |
| TC-09 | 시험지 미리보기 | 시험지 문항 선택 완료 | 미리보기 버튼 클릭 | 문제와 빈 답안 칸이 시험지 형식으로 표시 |
| TC-10 | PDF 출력 | 저장된 시험지 존재 | PDF 출력 실행 | PDF 파일 생성, 정답은 출력되지 않음 |
| TC-11 | 시험지 삭제 | 저장된 시험지 2개 이상 존재 | 두 번째 시험지 삭제 | 해당 exam_id 시험지만 삭제되고 목록 새로고침 |
| TC-12 | 직접 답안 입력 | 저장된 시험지와 학생 존재 | 학생별 답안 입력 후 저장 | exam_id + student_id 기준으로 답안 저장 |
| TC-13 | 학생 변경 답안 분리 | 학생 2명 이상 존재 | 학생 1 답안 저장 후 학생 2 선택 | 학생 1 답안이 학생 2 입력창에 남지 않음 |
| TC-14 | CSV 업로드 | 시험 문항 수와 학생 목록 존재 | 학번, 문항 답안 형식 CSV 업로드 | 학번 기준 학생 매칭 및 답안 파싱 |
| TC-15 | CSV 입력값 검증 | CSV 답안 업로드 완료 | 입력값 검증 실행 | 누락이 없으면 성공, 누락 시 학생/문항 안내 |
| TC-16 | 자동 채점 | 학생 답안 저장 완료 | 자동 채점 실행 | 정답 수, 오답 수, 점수, 정답률 계산 |
| TC-17 | 허용 답안 처리 | 허용 답안 등록 문제 존재 | 허용 답안과 일치하는 학생 답안 입력 | 정답 처리 |
| TC-18 | 답안 정규화 | 대소문자/문장부호 차이 답안 존재 | `Went.` 등 입력 후 채점 | 정규화 후 정답 처리 |
| TC-19 | 결과 보기 | 자동 채점 완료 | 결과 보기 실행 | 학생별 요약과 문항별 상세 표시 |
| TC-20 | 분석 요약 | 채점 결과 존재 | 분석 화면에서 시험/반 선택 | 평균 점수, 정답률, 오답률 정상 표시 |
| TC-21 | 전체 정답 예외 | 모든 문항 정답 | 분석 실행 | 오답률 0%, 취약 유형/피드백 미표시 |
| TC-22 | 대시보드 데이터 없음 | 채점 결과 없음 | 대시보드 진입 | 학생 수/문항 수 표시, 통계 0 또는 빈 리스트 |
| TC-23 | 대시보드 실제 데이터 | 채점 결과 존재 | 시험/반 선택 | 실제 학생 수, 문항 수, 차트, 학생별 결과 표시 |
| TC-24 | 대시보드 카드 매핑 | 6문제 중 1문제 정답 | 대시보드/분석 카드 확인 | 오답률은 83.33%, 평균 정답 수는 1.00 |
| TC-25 | CSV 형식 오류 | 잘못된 헤더 CSV 준비 | CSV 업로드 | 형식 오류 메시지 표시 |
