# 5. 테스트 계획

## 5.1 테스트 대상

| 테스트 대상 | 설명 |
| --- | --- |
| 학생 관리 | 학생 등록, 조회, 수정, 비활성화 및 기존 기록 보존 |
| 문제은행 관리 | 문제 등록, 검색, 수정, 비활성화, 정답/허용 답안 저장 |
| 시험지 생성 | 조건 기반 자동 추출, 직접 선택, 미리보기, 저장 |
| PDF 출력 | 생성된 시험지의 PDF 저장 여부 및 답안 미노출 확인 |
| 결과 입력 | 직접 입력, CSV 업로드, 입력값 검증, 학생별 답안 분리 |
| 자동 채점 | 정답 정규화, 기준 정답/허용 답안 비교, 점수 계산 |
| 결과 저장/조회 | 학생별 결과, 문항별 답안 기록, 시험별 결과 조회 |
| 결과 분석 | 반 평균, 정답률, 오답률, 취약 유형 분석 |
| 대시보드 | 실제 등록 학생/문항/채점 결과 기반 요약 카드, 차트, 학생별 표 |
| 예외 처리 | 데이터 없음, 문제 부족, CSV 형식 오류, 저장 실패 |

## 5.3 요구사항 검증 방법

| 검증 방법 | 적용 대상 | 설명 |
| --- | --- | --- |
| 단위 테스트 | Service, Repository, 유틸 함수 | 정답 정규화, 점수 계산, 분석 계산, 필터링 로직 검증 |
| 통합 테스트 | Controller-Service-Repository 흐름 | 저장, 조회, 채점, 분석까지 실제 DB 흐름 검증 |
| 화면 동작 테스트 | PySide6 View와 Controller 연결 | 버튼 클릭, 콤보박스 선택, 테이블 갱신, 메시지 표시 확인 |
| 데이터 검증 | SQLite DB | 저장된 students, questions, exams, answer_records, exam_results 확인 |
| 파일 출력 검증 | PDF, CSV | PDF 시험지 생성 및 CSV 업로드/내보내기 파일 확인 |
| 예외 테스트 | 입력 오류, 데이터 없음, 문제 부족 | 앱 종료 없이 안내 메시지와 안전한 기본값 반환 확인 |
| 코드 리뷰 | MVC 구조 | View/Controller/Service/Repository 역할 분리 확인 |
