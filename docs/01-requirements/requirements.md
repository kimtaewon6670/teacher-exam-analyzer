# 3. 요구사항

## 3.1 기능 요구사항

| ID | 기능 요구사항 | 우선순위 |
| --- | --- | --- |
| FR-01 | 학생을 등록할 수 있어야 한다. | 상 |

| FR-03 | 학생을 삭제하지 않고 비활성화할 수 있어야 한다. | 중 |
| FR-04 | 비활성화된 학생의 기존 시험 결과는 보존되어야 한다. | 상 |
| FR-05 | 단답형 문제를 문제은행에 등록할 수 있어야 한다. | 상 |
| FR-06 | 문제 유형은 어휘, 문법, 독해로 구분할 수 있어야 한다. | 상 |
| FR-07 | 문제별 세부 분류, 난이도, 태그, 해설을 저장할 수 있어야 한다. | 중 |
| FR-08 | 기준 정답과 허용 답안을 저장할 수 있어야 한다. | 상 |
| FR-09 | 문제 목록을 검색하고 유형, 난이도, 태그 등으로 필터링할 수 있어야 한다. | 상 |
| FR-10 | 문제를 수정하거나 비활성화할 수 있어야 한다. | 중 |
| FR-11 | 시험명, 설명, 시험 일자, 대상 반을 입력하여 시험지를 생성할 수 있어야 한다. | 상 |
| FR-12 | 유형별 문항 수와 난이도 조건을 기준으로 문제를 자동 추출할 수 있어야 한다. | 상 |
| FR-13 | 교사가 문제를 직접 선택하여 시험지에 포함할 수 있어야 한다. | 상 |
| FR-14 | 자동 추출 문제와 직접 선택 문제는 독립적으로 관리되고 저장 시 함께 반영되어야 한다. | 상 |
| FR-15 | 시험지 미리보기는 학생 배부용 형식으로 문항과 빈 답안 칸을 보여야 한다. | 상 |
| FR-16 | 시험지를 저장하고 저장된 시험지 목록을 조회할 수 있어야 한다. | 상 |
| FR-17 | 저장된 시험지를 고유 ID 기준으로 삭제할 수 있어야 한다. | 중 |
| FR-18 | 시험지를 PDF로 출력할 수 있어야 한다. | 중 |
| FR-19 | 학생별 답안을 직접 입력하고 저장할 수 있어야 한다. | 상 |
| FR-20 | CSV 파일을 통해 여러 학생의 답안을 불러올 수 있어야 한다. | 상 |
| FR-21 | CSV의 첫 번째 열 학번을 기준으로 학생을 매칭해야 한다. | 상 |
| FR-22 | 답안 입력값 검증 시 미입력 문항과 CSV 형식 오류를 확인해야 한다. | 상 |
| FR-23 | 학생 답안을 정규화하여 기준 정답 및 허용 답안과 비교해야 한다. | 상 |
| FR-24 | 학생별 정답 수, 오답 수, 점수, 정답률을 계산해야 한다. | 상 |
| FR-25 | 문항별 학생 답안, 기준 정답, 정답 여부를 저장해야 한다. | 상 |
| FR-26 | 시험별, 학생별 채점 결과를 조회할 수 있어야 한다. | 상 |
| FR-27 | 반 평균 점수와 전체 정답률을 계산해야 한다. | 상 |
| FR-28 | 문항별, 유형별, 세부 분류별, 난이도별 정답률과 오답률을 계산해야 한다. | 상 |
| FR-29 | 모든 문항을 맞힌 경우 취약 유형과 피드백은 표시하지 않아야 한다. | 중 |
| FR-30 | 대시보드는 실제 등록 학생, 시험 문항, 채점 결과를 기준으로 데이터를 표시해야 한다. | 상 |
| FR-31 | 데이터가 없는 경우 빈 목록 또는 0 값을 반환하고 앱이 오류 없이 동작해야 한다. | 상 |

## 3.2 비기능 요구사항

| ID | 비기능 요구사항 |
| --- | --- |
| NFR-01 | 앱은 Windows 환경에서 로컬 데스크톱 애플리케이션으로 실행되어야 한다. |
| NFR-02 | 데이터는 외부 서버 없이 로컬 SQLite 데이터베이스에 저장되어야 한다. |
| NFR-03 | View, Controller, Service, Repository, Model의 MVC 역할이 분리되어야 한다. |
| NFR-04 | DB 저장 및 조회는 Repository 계층에서만 처리해야 한다. |
| NFR-05 | 채점, 분석, 시험지 생성 핵심 로직은 Service 계층에서 처리해야 한다. |
| NFR-06 | Controller는 View와 Service/Repository 사이의 흐름 제어를 담당해야 한다. |
| NFR-07 | 답안 비교는 대소문자, 앞뒤 공백, 간단한 문장부호, 연속 공백을 정규화한 뒤 수행해야 한다. |
| NFR-08 | CSV 불러오기, 저장, 채점 중 오류가 발생하면 사용자에게 안내 메시지를 제공해야 한다. |
| NFR-09 | 기존 학생, 문제, 시험 결과 데이터는 비활성화 처리 후에도 조회 가능해야 한다. |
| NFR-10 | 대시보드와 분석 화면은 데이터가 없거나 채점 결과가 없어도 예외 없이 동작해야 한다. |

## 3.3 제약사항

| ID | 제약사항 |
| --- | --- |
| CON-01 | 웹 서비스가 아닌 로컬 데스크톱 앱으로 구현한다. |
| CON-02 | 학생이 앱에서 직접 시험을 응시하는 기능은 구현 범위에서 제외한다. |
| CON-03 | 모든 데이터는 사용자 PC 내부 SQLite DB에 저장한다. |
| CON-04 | 외부 서버와 외부 LLM API를 사용하지 않는다. |
| CON-05 | AI 의미 해석 채점은 사용하지 않는다. |
| CON-06 | 정답 비교는 선생님이 등록한 기준 정답과 허용 답안을 기준으로 한다. |
| CON-07 | 화면 디자인 변경은 UI 담당 범위이며 Core 작업에서는 View 디자인을 임의로 수정하지 않는다. |
| CON-08 | PDF 출력은 reportlab을 사용한다. |
| CON-09 | 그래프 시각화는 matplotlib 또는 PySide6 위젯 기반 차트로 구현한다. |
