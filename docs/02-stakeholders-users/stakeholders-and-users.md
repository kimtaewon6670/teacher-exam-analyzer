> 작성 기준: `feature/integration` 브랜치의 최종 통합 코드 구조와 기능 흐름

## 2.1 주요 사용자

| 구분 | 설명 | 주요 관심사 |
| --- | --- | --- |
| 교사 | Teacher Exam Analyzer의 실제 사용자이다. 학생, 문제은행, 시험지, 시험 결과를 관리한다. | 시험지 생성 시간 단축, 정확한 채점, 반별 취약 영역 파악 |

## 2.3 사용 시나리오

### 시나리오 1. 문제은행 기반 시험지 생성

1. 교사가 문제은행 관리 화면에서 단답형 문제를 등록한다.
2. 문제 유형, 세부 분류, 난이도, 기준 정답, 허용 답안, 해설, 태그를 입력한다.
3. 시험지 생성 화면에서 시험명, 시험 일자, 대상 반을 입력한다.
4. 유형별 문항 수와 난이도 조건을 설정하거나 문제를 직접 선택한다.
5. 시험지 미리보기로 문항 구성과 답안 작성 칸을 확인한다.
6. 시험지를 저장하고 필요 시 PDF로 출력한다.

### 시나리오 2. 시험 결과 입력 및 자동 채점

1. 교사가 결과 입력 화면에서 시험, 반, 학생을 선택한다.
2. 학생별 답안을 직접 입력하거나 CSV 파일을 불러온다.
3. CSV 파일은 학번 열과 문항 번호 열을 기준으로 현재 반의 학생과 매칭된다.
4. 입력값 검증을 실행하여 미입력 문항, 학생 매칭 실패, 형식 오류를 확인한다.
5. 답안을 저장하고 자동 채점을 실행한다.
6. 같은 학생을 재채점하는 경우 기존 결과와 답안 기록은 exam_id + student_id 기준으로 교체 저장된다.
7. 시스템은 기준 정답 및 허용 답안과 비교하여 정답 여부를 판단한다.
8. 학생별 정답 수, 오답 수, 점수, 정답률을 저장한다.

### 시나리오 3. 시험 결과 분석

1. 교사가 결과 분석 또는 대시보드 화면으로 이동한다.
2. 시험과 반을 선택한다.
3. 시스템은 저장된 채점 결과를 기반으로 통계를 계산한다.
4. 교사는 반 평균 점수, 전체 정답률, 문항별 정답률, 유형별 정답률을 확인한다.
5. 교사는 대시보드에서 학생별 결과를 검색, 페이지 이동, 전체 보기, CSV 내보내기로 확인할 수 있다.
6. 교사는 취약 유형과 세부 분류를 확인하여 보충 수업 계획에 활용한다.
